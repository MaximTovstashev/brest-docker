What the hell do I see?

This is REST API description document, built automatically. The documentation is being automatically rebuilt each time the server is restarted.

While methods, URI files and validation is parsed from API files, the descriptions are added by hand, which means that they can be outdated or otherwise misguiding. Please note that in case of doubt and contact the developer for up-to-date info.

What can I do?

For each API resource add "description" property. It can be either string of multiline hereDoc string. Both variants unilize MarkDown syntax

Since markdown depends on line breaks in several ways, the hereDoc is used to form multiline string constants.


    description: function() {/*
        #Here's the sample

        Of multiline hereDoc constant. Yes, it's within the comment, within the function.
        However, it's powered by dozens of magic Unicorns and therefore works.

        You can use *markDown* here.
    */}
